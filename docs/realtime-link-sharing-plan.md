# 커피내기/룰렛 링크공유 실시간 동기화 가능 여부

결론부터 말하면 **가능합니다**.
현재 저장소에는 1차 구현(방 생성/입장 + 실시간 구독)이 반영되어 있으며, 아래를 기준으로 운영/고도화하면 됩니다.

## 1) 필수 구성요소
- Room 개념 (`roomId`)
- 중앙 상태 저장소 (Supabase `rooms.game_state`)
- 실시간 구독 (Supabase Realtime)

## 2) 현재 반영된 동작
1. Host가 방 생성
2. `?roomId=...&role=viewer` 링크 공유
3. 참여자는 링크로 입장
4. 모든 클라이언트가 같은 `game_state`를 구독
5. Host 액션(단계 전환, 값 변경)을 DB 업데이트
6. 참여자는 동일 화면으로 즉시 반영

## 3) 기존 커피내기/룰렛 적용 원칙
- 게임 원본 흐름을 유지하고 상태 동기화 레이어를 상단 페이지에 추가
- Viewer는 입력 불가(읽기 전용)
- Host가 상태를 기록하고 Viewer는 그 상태를 렌더링

## 4) 다음 우선순위
1. 커피내기 결과 페이지도 room 상태와 연동
2. 룰렛 회전 완료 결과를 room 상태에 기록
3. RLS 정책/만료 정책(오래된 방 정리) 적용
4. 재접속/오프라인 복구 UX 개선

## 5) 리스크
- 지연으로 인한 화면 순간 불일치
- 동시에 입력했을 때 충돌
- 모바일 브라우저 백그라운드 복귀 시 구독 끊김

## 6) 성공 기준
- 링크 접속한 2대 이상 기기에서 동일한 단계/값/결과 유지
- Host 조작 후 1초 내 동기화
- 재접속 후 room 최신 상태 복구
